# Design Document: C.I.V.I.C. 

**C.I.V.I.C. (Community Infrastructure Verification & Intelligent Clustering)** is a mobile-first civic infrastructure tracking platform built using React, Tailwind CSS, Node.js (Express), Cloud Firestore, Firebase Authentication, and Google Maps Platform.

## High-Level Architecture

The architecture is a full-stack, AI-enhanced application designed to facilitate civic issue reporting and resolution.

1.  **Frontend (React & Tailwind):** 
    -   Provides the user interface for capturing images, displaying the gamification dashboard, and rendering the interactive map.
    -   Tailwind CSS is used for rapid, utility-first styling ensuring responsive and accessible design.
2.  **Mapping Layer (Google Maps API):**
    -   Integrates interactive maps to plot reported issues using custom styled markers based on severity.
3.  **Backend (Node.js/Express):**
    -   A lightweight custom Express server handles requests from the client.
    -   It orchestrates the AI pipelines, validates payloads, and serves as a secure proxy to the Gemini API, keeping API keys hidden from the client.
4.  **Database (Firebase Firestore):**
    -   A NoSQL, real-time database that stores citizen profiles (points, ranks), issues, and corroborated reports. 
    -   Uses real-time snapshots (`onSnapshot`) to push updates back to the client immediately.
5.  **AI Layer (Gemini API):**
    -   Google's `gemini-2.5-flash` model performs multimodal analysis of user-submitted images, structuring raw visual data into structured JSON output.

## Agentic Pipelines

The server (`server.ts`) handles incoming images which are passed through modular pipelines to the Gemini API for robust analytics:

-   **Triage Agent**: Conducts multimodal image categorization. It receives a picture of the issue and uses Gemini Multimodal Structured Outputs to generate a JSON object containing: `category`, `severityScore` (1-10), `confidencePercentage`, `recommendedDepartment`, and `autoGeneratedDescription`. This transforms raw images into actionable civic data.
-   **Routing Agent**: Applies deterministic rules based on the Triage Agent's `confidencePercentage`. If confidence is high (> 85%), it automatically routes the issue (`Auto-Routed`). Otherwise, it flags it for `Requires Human Verification`.
-   **Verification Agent**: Operates spatially by calculating the Haversine distance between the newly reported issue and existing unresolved issues. If a similar issue is reported within a 50-meter radius of the exact same category, it corroborates the report, merging it into an active group and incrementing importance.

## File Structure & Architectural Patterns

```text
/
├── server.ts                  # Main Express backend, controls AI Pipelines & routing.
├── server/
│   └── gemini.ts              # Agentic logic for Triage, Routing, and Verification.
├── src/
│   ├── components/            
│   │   ├── Reporter.tsx       # UI for image capture, geolocation, and AI interaction.
│   │   ├── ImpactDashboard.tsx# UI for gamification, leaderboards, and user points.
│   │   └── CommandMap.tsx     # Fullscreen Google Maps visualization.
│   ├── lib/                   
│   │   └── firebase.ts        # Client-side Firebase configuration and utility functions.
│   ├── types.ts               # Shared TypeScript interfaces (CivicIssue, CitizenProfile, etc.)
│   ├── App.tsx                # Main React orchestrator & Authentication router.
│   └── index.css              # Global styles & Tailwind configuration.
```

### Significant Architectural Patterns

-   **Agentic Workflow**: Breaking down complex analysis into specialized agent sub-tasks (Triage -> Routing -> Verification) keeps the code robust, modular, and easy to maintain.
-   **Client-Server AI Proxy Pattern**: The Gemini API key remains solely on the Express server. The React client explicitly calls `/api/triage`, mitigating client-side secret exposure and handling intensive tasks server-side.
-   **Real-time Synchronization**: By using Firebase `onSnapshot` in React Hooks (`useEffect`), the client maintains a constant, live representation of civic infrastructure data without needing to poll the server.
